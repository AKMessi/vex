# HyperFrames Visual World Compiler

## Objective

Automatic visuals must preserve source-backed meaning without collapsing every
explanation into the same background, card system, and connector grammar.

Vex now separates semantic truth from expressive realization:

1. subtitles and evidence compile into Visual Explanation IR
2. the signed claim graph and proof tournament preserve the explanation
3. the Visual World compiler selects an appropriate visual medium
4. a specialized deterministic backend compiles that medium to HyperFrames
5. semantic, visual, perceptual, and portfolio-level QA decide promotion

## Signed Visual World Program

Every proof candidate receives a signed program containing:

- medium family
- canvas and background system
- shape and material language
- typography system
- camera depth
- motion choreography
- card policy and panel-area target
- semantic object and relation bindings
- palette
- visual fingerprint

Changing any field invalidates the program signature. Expressive choices cannot
silently detach themselves from the signed Scene Program V2 contract.

## Renderer Backends

The first production backends are:

- kinetic typography
- editorial collage
- data sculpture
- spatial metaphor
- diagrammatic technical system
- product interface
- grounded source-media composite

Cards are forbidden for typography, collage, sculpture, and metaphor scenes.
Product surfaces remain available when the subject is an actual interface.

Every backend preserves stable object, relation, fact, evidence, scene-program,
and proof-program identifiers in the generated composition.

## Video Design Bible

Auto Visuals builds one signed design bible for primary and reserve
opportunities. It coordinates:

- palette progression
- typography continuity
- recurring registration motifs
- maximum card-area policy
- rolling medium and background repetition limits

Candidate compilation receives the recent accepted visual-world history.
Continuity comes from deliberate type and color systems, not identical layouts.

## Dual Search

The proof tournament now searches two independent dimensions:

- semantic proof encoding
- visual-world medium

Candidates must differ in how they explain and in how they look. Cosmetic-only
palette variants do not satisfy the diversity objective.

## Perceptual Portfolio QA

Rendered frames produce a deterministic fingerprint containing:

- color histogram
- mean RGB
- luminance and contrast
- saturation
- edge density
- medium, background, and motion metadata

The creative-set optimizer rewards distinct media, canvases, backgrounds, and
motion grammars. Final timeline QA rejects duplicate fingerprints and
perceptually similar neighboring visuals that reuse the same visual world.

## Runtime And Release

The repository and managed pipx runtime pin HyperFrames `0.6.113`. Release
validation requires the Visual World compiler, specialized renderer, native
generated-video motion compiler, perceptual QA, and creative optimizer in both
wheel and source distribution artifacts.
