# Semantic Blueprints

Select by explanation type and grounded object roles. Art direction is secondary.

## Scene Families

| Scene type | Required idea | Preferred devices |
| --- | --- | --- |
| `metric_delta` | measured before/after change | registered states, metric bridge, evidence axis |
| `metric_intervention` | measured change caused by an intervention | intervention gate, tracked metric, cause trace |
| `metric_proof` | one measured claim with evidence | evidence geometry, proof marker, metric lock |
| `causal_intervention` | problem through mechanism to outcome | causal spine, mechanism chamber, intervention gate |
| `guided_process` | ordered steps or handoff | route, persistent traveler, completion trail |
| `architecture_flow` | service lifecycle | boundaries, request token, return path |
| `matched_state_transform` | inspectable state change | matched geometry, fixed constraint anchor |
| `grounded_interface_walkthrough` | named action and result | stable UI surface, focus trace, feedback path |
| `decision_branch` | named condition with grounded outcomes | decision gate, exclusive branch activation |
| `narrative_progression` | setup, decisive turn, payoff | persistent subject, trajectory change |
| `evidence_backed_quote` | exact retained language | phrase assembly, clause focus, full quote return |

## Selection Rules

- Require every blueprint role before selection.
- Prefer a specialized variant when source language matches its tags.
- Keep at least two curated variants per scene type.
- Store rationale, layout thesis, motion spine, devices, and anti-patterns with each blueprint.
- Reject instead of adapting a blueprint whose prerequisites are absent.

## Legacy Templates

Keep legacy names for manual projects and compatibility. Do not use them as the automatic semantic planning vocabulary. New automatic work should compile to `semantic_*` stage families.
