# Typed Counterexamples

Every issue must include:

- critic identity: blind, grounded, or design
- fixed issue type and severity
- frame ID and timestamp when available
- existing element IDs, relation IDs, or normalized regions
- expected versus observed behavior
- evidence IDs
- confidence
- allowed repair operations

Reject unknown IDs, unknown issue types, unknown repairs, and prose-only feedback.

Blind criticism receives frames without the intended answer. Grounded criticism receives
the signed contract and source context. Design criticism evaluates hierarchy, density,
typography, geometry, pacing, and professional finish.
